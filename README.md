# HDServerConfig

Утилита командной строки (Standalone C++) для настройки LED-контроллеров Huidu (Android BoxPlayer серия, протестировано на HD-A3L) через TCP сокеты (SDK 3.0).

> [!IMPORTANT]
> **Архитектурный прорыв (Android-версии):**
> Абсолютное большинство документации Huidu указывает на порт `10001` для XML SDK команд. Однако, контроллеры серий Android (AxL, A7, A8, такие как **HD-A3L**) резервируют порт 10001 исключительно для своего внутреннего недокументированного потока (JSON API модуля HDPlayer). 
> 
> **Официальный XML SDK порт для этих устройств — `20001`**. 
> Любые попытки отправки XML-команд (GetDeviceInfo, SetSDKTcpServer и т.д.) на порт 10001 будут отклонены с ошибкой `kUnsupportMethod`.

## Технические детали

Программа реализует **ручной независимый цикл общения** по спецификации Huidu SDK 3.0 без использования проприетарной `HDSDK.dll` (типы данных интегрированы напрямую):

1.  **Binary Handshake (0x2001)**: 8-байтовый пакет запроса версии (v0x1000007).
2.  **XML Packet Header (0x2003)**: Структурный `HTcpExt` (12 байт: `Len`, `Cmd=0x2003`, `TotalSize`, `Index`).
3.  **Strict GUID Auth (Уязвимость сессии)**:
    *   Контроллер выдает токен через команду `GetIFVersion`.
    *   **ВНИМАНИЕ**: Android-серии (порт 20001) чувствительны к регистру. GUID должен передаваться строго байт-в-байт без дефисов в **нижнем регистре** (например, `f6c1dedff5...`).
    *   Принудительное приведение GUID к верхнему регистру (`ToUpper()`) или изменение формата приведет к бинарному отказу контроллера — **Ошибка протокола 45 (`kGuidNotMatch` / `kSessionInvalid`)**.

## Использование

### Проверка настроек и параметров устройства
```powershell
.\HDServerConfig.exe verify 192.168.8.2
```

### Установка адреса Cloud Server
```powershell
.\HDServerConfig.exe set 192.168.8.2 77.42.66.37 10001
```

### Демонстрационная передача программы
```powershell
.\HDServerConfig.exe add_program 192.168.8.2
```

### Удаленная перезагрузка
```powershell
.\HDServerConfig.exe reboot 192.168.8.2
```
